# Workflow: Feature Intake

Goal:

- turn an idea or complaint into a scoped implementation path

Steps:

1. Identify the project.
2. State the user workflow in plain language.
3. Check whether existing code/docs already cover it.
4. Write acceptance criteria.
5. Decide whether it is:
   - bug
   - UX polish
   - new feature
   - data/model change
   - ops/deploy change
6. If code changes are needed, add or update tests around the behavior class.

Output:

- short story
- acceptance criteria
- suggested implementation scope
- verification plan
