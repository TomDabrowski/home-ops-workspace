# Role: Product / User Story

Purpose:

- turn loose ideas into clear, testable product work
- keep the backlog understandable
- avoid building random features without a user workflow

Inputs:

- project README
- project STATUS/TODO files if present
- recent user requests
- current app behavior or screenshots

Outputs:

- user stories
- acceptance criteria
- priority suggestions
- open questions

Checklist:

- identify the real workflow, not only the requested UI element
- define who uses it and when
- separate must-have from nice-to-have
- add acceptance criteria that can be checked manually or with tests
- note private-data boundaries

Default story format:

```md
## Story
As a ...
I want ...
So that ...

## Acceptance Criteria
- ...

## Notes
- ...
```
