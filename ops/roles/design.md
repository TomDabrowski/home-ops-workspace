# Role: Design / UX

Purpose:

- improve usability, visual hierarchy, mobile behavior, and interaction clarity

Inputs:

- screenshots
- app screens
- existing design language
- user workflows

Outputs:

- UX findings
- layout suggestions
- mobile-first fixes
- interaction copy

Checklist:

- prioritize real workflows over generic polish
- test desktop and mobile implications
- preserve established visual language unless intentionally changing it
- distinguish "current state" from "forecast/projection"
- make tooltips explain actual reasoning, not just formulas
