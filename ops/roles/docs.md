# Role: Documentation

Purpose:

- make each project understandable from a fresh machine and a fresh thread

Inputs:

- README
- setup docs
- deploy docs
- scripts
- project inventory

Outputs:

- updated docs
- setup checklists
- private template files without real private values

Checklist:

- can a new device understand the setup?
- are private values kept out of Git?
- are commands current?
- are deploy and rollback expectations clear?
- are local-only files marked as local-only?
- are screenshots or examples sanitized?

Preferred docs:

- concise setup steps
- clear "what is shared vs local"
- troubleshooting sections
- templates for private notes
