## Working Rules

- Keep each tool as its own project folder or repo.
- Store current state inside each project in STATUS.md.
- Use README.md for scope, MVP, and setup notes.
- Use `ops/project-inventory.yaml` for cross-project agent context and priorities.
- Use `ops/platform-architecture.md` for the broader personal DevOps / NAS runtime model.
- Keep any filled all-repo registry outside Git.
- Push to Git after meaningful milestones, not after every tiny edit.
- Keep raw finance exports and generated analysis files untracked by default.
